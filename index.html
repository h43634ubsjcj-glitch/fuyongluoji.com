安装第三方App

AIbangOS 原生不支持安装第三方应用，所有安装均需通过系统安全漏洞实现。根据系统版本选择对应方案。

向下选择
01 / 02
📡

AIbangOS ≤ 6.2.0

适用于 6.2.0 及以下所有版本。利用应用商店在代理环境下仍可下载的漏洞，通过抓包工具拦截请求、替换下载链接，实现安装第三方 App。

教程包含所需工具、配置步骤和完整操作流程。
02 / 02
🐛

AIbangOS < 3.8

适用于 3.8 以下版本。利用包名大小写漏洞，无需授予「未知来源」权限，输入本地 APK 路径即可安装。3.8 及以上已修复。

提供 JC Code V2（修改版）下载及使用说明。
所有方式均基于系统漏洞，请确保 App 来源可信。若跳转无反应，请使用推荐浏览器：Via、悟空、头条、夸克。

INTENT TEST
关于本机 跳转测试

逐个测试哪种方案能在你的设备上正常跳转。请打开 ADB 日志抓取报错信息。

向下测试
01 / 05
📋

方案一：Fragment 跳转

使用 ZybSettings + MyDeviceInfoFragment，标准 Fragment 分发入口。

通用入口 - JS
通用入口 - YS
通用入口-JS使用 .ZybSettings
通用入口-YS使用 .zyb.ZybSettings
component=com.android.settings/.ZybSettings
fragment=com.android.settings.deviceinfo.MyDeviceInfoFragment
02 / 05
🔀

方案二：SubSettings 入口

使用 SubSettings 作为通用容器入口，搭配同一个 Fragment。两个版本 URI 相同。

通用入口 - JS
通用入口 - YS
通用两个版本 URI 相同，选任意一个即可
component=com.android.settings/.SubSettings
fragment=com.android.settings.deviceinfo.MyDeviceInfoFragment
03 / 05
⚡

方案三：Activity 直跳

直接调用 Settings$MyDeviceInfoActivity，绕过 Fragment 分发机制，不区分版本。

component=com.android.settings/.Settings$MyDeviceInfoActivity
04 / 05
📦

方案四：完整 Intent 参数

在方案三基础上增加 action 和 category，模拟系统快捷方式的完整 Intent。

action=android.intent.action.MAIN
category=com.android.settings.SHORTCUT
component=com.android.settings/.Settings$MyDeviceInfoActivity
05 / 05
🔍

方案五：SETTINGS Action

使用 com.android.settings.action.SETTINGS action 配合 Fragment 参数，模拟搜索功能的跳转方式。

action=com.android.settings.action.SETTINGS
fragment=com.android.settings.deviceinfo.MyDeviceInfoFragment
测试时请打开 ADB 日志抓取，把报错信息发给开发者分析。推荐使用 Via、悟空浏览器、今日头条、夸克 进行测试。
